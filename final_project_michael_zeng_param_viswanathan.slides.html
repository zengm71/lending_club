W207 Final Project:
Lending Club Loan Data Default Classification

Michael Zeng, Param Viswanathan

Introduction¶

  • The data set includes more than 1 million loans that Lending Club issued from 2008 to 2018, with various features of the loans.

  • Our goal is to analyze the loan data and train a model to identify if the next loan applicant will default or not.

  • This study will be very helpful to screen high risk applications as well as monitoring the risk of current portfolio.

  • We will first do EDA on the data to understand it better and then decide on how to clean up the data and also transform any values for certain features.
  • We will divide the data set into training, developing and testing, while testing set will be held out for the final comparison.
  • We will be evaluating various machine learning classifiers:
    • Logistic Regression
    • KNN with PCA
    • SVM
    • Random Forest
    • Naive Bayesian with PCA
    • Neural Network

Data Cleaning¶

The data is updated 4 months ago. For loans issued recently that are still in term (not yet defaulted), we are not sure if they are going to default. So the first thing to do is to remove them so that we are left with a data set with reliable labels.

In [23]:
# the data set is as of 4 months ago
df['loan_age_months'] = df.issue_d.apply(lambda x: (datetime.now() - datetime.strptime(x, '%b-%Y')).days / 30) - 4
df['term_months'] = (df.term == ' 36 months') * 36 + (1 - (df.term == ' 36 months')) * 60
sum(df.term_months > df.loan_age_months) / df.shape[0]
Out[23]:
0.6347017784123985

This shows that about 63% of the loans are still within their terms. For these loans, unless they have already defaulted, we can't really trust the labels. We will remove the loans that might still default from our modeling dataset df_m.

In [26]:
bad_loan_status = ["Charged Off", "Default", "Does not meet the credit policy. Status:Charged Off", 
                   "In Grace Period", "Late (16-30 days)", "Late (31-120 days)"]
might_still_default = (df.loan_status == 'Current') & (df.term_months > df.loan_age_months)

# data frame for modeling
df_m = df[~ might_still_default]
df_m.shape[0] / df.shape[0]
Out[26]:
0.605752812885395

We are now left with about 60% of the original observations, for which we know the labels are trustworthy.

from sklearn.neighbors import KNeighborsClassifier

def KNN(k_values):
    for k in k_values:
        k_neighbor_clf = KNeighborsClassifier(k, n_jobs=5)
        k_neighbor_clf.fit(X_train_projected, y_train)
        
        predict_val = k_neighbor_clf.predict(X_val_projected)
        accuracy_val = metrics.accuracy_score(y_pred= predict_val, 
                                                y_true = y_val)
        f1_val = metrics.f1_score(y_pred= predict_val, y_true = y_val)
        
        if k == 1:
            tuning = pd.DataFrame([[k, 
                                    "%0.2f%%" % (accuracy_val * 100), 
                                    "%0.2f%%" % (f1_val * 100)]], 
                                  columns= ['K', 'Accuracy Val', 'F1 Val'])
        else:
            tuning = tuning.append(pd.DataFrame([[k, 
                                                  "%0.2f%%" % (accuracy_val * 100), 
                                                  "%0.2f%%" % (f1_val * 100)]], 
                                  columns= ['K', 'Accuracy Val', 'F1 Val']), 
                                  ignore_index=True)
    print(tuning)

  • we experimented with some of the most popular models and tested their suitability for dealing with unbalanced and high dimensional classifications;
